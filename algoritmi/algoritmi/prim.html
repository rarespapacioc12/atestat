Algoritmul lui Prim

Algoritmul lui Prim este folosit pentru a construi arborele de acoperire minimă (MST) al unui graf ponderat conex. MST este un subgraf fără cicluri care conectează toate vârfurile la cost minim total.

Descriere

Prim începe de la un vârf arbitrar și extinde arborele alegând muchia de cost minim care duce la un nod încă neinclus.

Varianta clasică O(n^2) (folosind matrice de adiacență)

function PrimMatrix(n, cost):
    inițializează distanțele cu infinit
    inițializează vizitat[] = false
    distanța[start] = 0
    vizitat[start] = true

    pentru i = 1 la n:
        alege vârful nevizitat cu distanța minimă, fie curent acest nod
        vizitat[curent] = true
        pentru fiecare vârf v:
            dacă există muchie (curent, v), v nu e vizitat și costul e mai mic:
                actualizează distanța[v]

Varianta optimizată O((n + m) log n) (folosind heap)

function PrimHeap(n, lista_adiacenta):
    initializează distanțele cu infinit
    initializează vizitat[] = false
    creează heap cu (0, start)

    while heap nu este gol:
        extrage minim (cost, v)
        dacă v este deja vizitat:
            continuă
        marchează v ca vizitat
        pentru fiecare vecin (u, costu) al lui v:
            dacă u nu este vizitat și costu < distanță[u]:
                distanță[u] = costu
                adaugă (distanță[u], u) în heap

Explicații

În varianta pătratică căutăm manual minimul la fiecare pas. În varianta cu heap (priority queue), putem găsi și actualiza rapid costurile minime.

Exemplu de aplicare

Graful:

A -1- B
|    |
4    2
|    |
C -1- D

Prim poate construi arborele A-B-D-C cu cost total 4.

Complexitate

  • O(n^2) cu matrice de adiacență
  • O((n + m) log n) cu heap (pentru grafuri rare)
