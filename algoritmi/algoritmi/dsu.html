Disjoint Set Union (DSU)

Structura de date Disjoint Set Union (DSU) sau Union-Find este utilizată pentru a gestiona o colecție de mulțimi disjuncte, cu operații eficiente de reuniune și apartenență.

Operații separate

Inițializare

function Initialize(n):
    pentru fiecare vârf i:
        parinte[i] = i
        rang[i] = 0
        dimensiune[i] = 1

Find cu compresie de drum

function Find(x):
    dacă parinte[x] != x:
        parinte[x] = Find(parinte[x])
    return parinte[x]

Union după înălțime (rang)

function UnionByHeight(x, y):
    rx = Find(x)
    ry = Find(y)
    dacă rx == ry:
        return
    dacă rang[rx] < rang[ry]:
        parinte[rx] = ry
    altfel dacă rang[rx] > rang[ry]:
        parinte[ry] = rx
    altfel:
        parinte[ry] = rx
        rang[rx] += 1

Union după dimensiune (size)

function UnionBySize(x, y):
    rx = Find(x)
    ry = Find(y)
    dacă rx == ry:
        return
    dacă dimensiune[rx] < dimensiune[ry]:
        parinte[rx] = ry
        dimensiune[ry] += dimensiune[rx]
    altfel:
        parinte[ry] = rx
        dimensiune[rx] += dimensiune[ry]

Explicații

Initialize: Setează fiecare nod să fie părintele său, cu rang 0 și dimensiune 1.

Find: Aplică "path compression" pentru a accelera căutările ulterioare.

UnionByHeight: Atașează arborele cu rang mai mic la cel cu rang mai mare pentru a menține arborii jos.

UnionBySize: Atașează arborele mai mic (ca număr de noduri) la cel mai mare, pentru echilibrare optimă.

Exemplu de aplicare

Inițial:

1  2  3  4  5

Apelăm UnionByHeight(1, 2) → 1 și 2 în aceeași mulțime.
Apelăm UnionBySize(3, 4) → 3 și 4 în aceeași mulțime.
Apelăm UnionByHeight(2, 3) → 1, 2, 3, 4 în aceeași mulțime.

Complexitate

  • Operații Find și Union: aproape O(1) timp amortizat (O(α(n))) (unde α reprezintă inversul funcției Ackermann).
  • Complexitate spațiu: O(n)
